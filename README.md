# 思考行動知識OS Doc化 GASツール

## 1. 初期設定手順

1. Googleスプレッドシートを開き、上部メニューから `[拡張機能] > [Apps Script]` をクリックします。
2. 開いたGASエディタを利用するか、本フォルダにある `.gs` ファイルをclasp等でアップロードします。
   手動の場合は、左側の「ファイル（＋アイコン）」から「スクリプト」を選択し、以下の6つの空ファイルを作成します。
   - `main`
   - `config`
   - `sheet`
   - `drive`
   - `log`
   - `utils`
   （※初期からある `コード.gs` は削除するか名前変更しても構いません）
3. 各ファイルに、本フォルダ内の各ソースコードをそれぞれコピーして貼り付け、上部にある🖬（保存）アイコンを押して保存します。
4. ドキュメントを作成するためのGoogle Driveのフォルダを3つ（思考用、行動用、知識用）作成し、それぞれの「フォルダID」を控えておきます。
   *(※フォルダのURL `https://drive.google.com/drive/folders/XXXXXXXXXX` の `XXXXXXXXXX` の部分)*

---

## 2. configシート作成手順

スプレッドシートにて、新しくシートを作成し、シート名を半角英小文字で `config` としてください。
1行目にヘッダを書き、2行目以降に設定値を入力します。

| A列(key) | B列(value) | C列(備考・自由記述) |
|---|---|---|
| key | value | 備考 |
| target_sheet_name | 日記 | 処理対象とするシート名 |
| thought_folder_id | (控えた思考OSフォルダID) | 思考OS用保存先 |
| action_folder_id | (控えた行動OSフォルダID) | 行動OS用保存先 |
| knowledge_folder_id | (控えた知識OSフォルダID) | 知識OS用保存先 |
| status_target | 未 | 処理対象ステータス |
| status_done | 済 | 処理完了ステータス |
| trigger_enabled | TRUE | 定期実行を有効にするか（TRUE/FALSE） |
| trigger_hour | 5 | トリガー稼働予定時間（0〜23）機能メモ |
| log_sheet_name | log | ログ出力先シート名 |
| header_date | 日付 | ヘッダ名 |
| header_thought | 思考OS | ヘッダ名 |
| header_action | 行動OS | ヘッダ名 |
| header_knowledge | 知識OS | ヘッダ名 |
| header_doc_status | Doc化 | ヘッダ名 |
| header_thought_url| 思考DocURL | ヘッダ名 |
| header_action_url | 行動DocURL | ヘッダ名 |
| header_knowledge_url| 知識DocURL | ヘッダ名 |

---

## 3. logシート作成手順

スプレッドシートにて、新しくシートを作成し、シート名を半角英小文字で `log` としてください。
1行目（A〜I列）に以下のヘッダを入力してください。（GASはこのシートへ追記します）

1. A1: `実行日時`
2. B1: `対象シート`
3. C1: `行番号`
4. D1: `日付`
5. E1: `結果`
6. F1: `詳細`
7. G1: `思考DocURL`
8. H1: `行動DocURL`
9. I1: `知識DocURL`

---

## 4. 実行方法

### 4-1. 手動実行方法
1. スプレッドシートの画面を再読み込み（F5更新）します。
2. 上部メニューに **`OS Doc化`** というメニューが追加されます。
3. まず初めに **`OS Doc化` > `設定チェック`** をクリックします。
   （※初回実行時は「承認が必要です」というダイアログが出るので、「続行」→自身のアカウント選択→「詳細を表示」→「（安全でないページ）へ移動」→「許可」を行ってください）
4. 設定チェックが「完了」と出れば準備OKです。
5. **`OS Doc化` > `ドキュメント作成実行`** をクリックすると、対象行の処理が開始され、終了後に結果サマリダイアログが表示されます。

## 5. トリガー設定方法（定期実行）

毎日自動でドキュメント作成を走らせる設定です。

1. **`OS Doc化` > `トリガー設定案内`** をクリックして手順の概要を確認できます。
2. `[拡張機能] > [Apps Script]` からエディタを開きます。
3. 画面左側の **時計アイコン（トリガー）** をクリックします。
4. 右下の **「トリガーを追加」**（青いボタン）をクリックします。
5. 以下の通りに設定します。
   - 実行する関数を選択: **`executeTrigger`**
   - 実行するデプロイを選択: `Head`
   - イベントのソースを選択: **`時間主導型`**
   - 時間ベースのトリガーのタイプを選択: **`日付ベースのタイマー`**
   - 時刻を選択: （例：`午前5時〜6時` など、`config`の想定時間に合わせて選択）
6. **保存** ボタンを押します。（承認画面が出たら許可します。）

※ スクリプト内で `config` シートの `trigger_enabled` が `TRUE` であるかを判定しています。一時的に止めたい場合は、トリガーを消さなくても `config` の値を `FALSE` に変えるだけで定期実行はスキップされます。

## 6. 想定エラーと対処法一覧

| エラーの事象（または表示されるエラー文） | 原因 | 対処法 |
| --- | --- | --- |
| **configシートが見つかりません** | configシートが存在しないか、シート名が間違っている。 | シート名が半角英小文字でぴったり `config` であるか確認してください。（スペースなどが混ざっていないか） |
| **〇〇シートに必須のヘッダが存在しません** | 日記シート上に、configで指定したヘッダ名と同じ列が見つからない。 | configシートの列設定値と、実際の日記シート1行目の列名が **「一言一句」** 同一であるか確認してください。（全角半角の違い、スペースの有無など） |
| **フォルダIDへのアクセス失敗権限がありません** | configの `xxxx_folder_id` が間違っているか、削除されたフォルダを指定している。 | ドライブを開き、対象フォルダのURLの `folders/` 以下の文字列を正しくコピペし直してください。 |
| **起動してもずっと「スクリプトを実行しています」のまま終わらない** | 対象「未」行が膨大で、Google Apps Scriptの実行制限時間（6分）を超過している。 | 手動で数行分だけ「済」に変更して対象を減らすか、次回実行時（途中で止まってもURL埋まり状況は保持／冪等性）に続きから再開するのをお待ちください。 |
| **logシートの結果が FAIL (日付データのパースに失敗)** | [日記]シートの「日付」列に、`2026/02/31` など実在しない日や、純粋なテキストが入力されている。 | 該当行のA列日付を、カレンダー形式で正しい日付に修正してください。 |
| **logシートの結果が FAIL (Exceeded maximum execution time)** | Google Document作成のAPI呼び出しなどが重なり、1行の処理途中でGASの制限時間を迎えた。 | URL列が埋まった種別分は次回スキップされるため、気にせず再実行（「ドキュメント作成実行」）してください。 |
| **メニューに「OS Doc化」が出てこない** | `main.gs` 内の `onOpen` 関数が正しく保存されていないか、スプレッドシートの読込が不完全。 | GASエディタでファイルを確実に保存し、スプレッドシートのブラウザタブを再読み込み(F5)してください。 |
